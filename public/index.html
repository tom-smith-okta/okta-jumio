Okta and NGINX

Integrate Okta identity management with NGINX application access and NGINX API gateway for the ultimate in security and ease-of-use, for admins, developers, and end-users.

To walk through the demo, follow the steps below. To see the resources available, click on the resources icon in the upper right.

Visit the downstream app, unproxied: downstream app

The downstream app will show some headers. Not very exciting.

Remember the following credentials: lois.lane / Okta1234$

Visit the NGINX+ proxy: NGINX+

Or

Visit the NGINX oss proxy: NGINX oss

The NGINX server redirected you to authenticate against Okta. You authenticated as Lois Lane, and saw the new headers that NGINX passed to the downstream app. NGINX parsed these values from the Okta id token. Sweet.
